# 审查压缩——实证数据

2026-03-14

---

## 摘要

设计文档（*Review scaling and complexity* 一节）声称每项的审查面
"通常约为代码面的 10%"。本文档呈现了来自 liyi 代码库自身的首次实证
测量——一项自参照数据：工具自身的代码已被完整标注了意图规格。

**核心结论：审查者只需阅读约 394 行意图描述，即可替代阅读约 10,488
行源代码——审查面缩减 26.6 倍。** 纯意图文本仅占源代码行数的 3.8%。

---

## 方法论

由测量脚本 `scripts/measure-intent-ratio.py` 遍历代码库并收集以下
数据：

1. **源文件** — `crates/` 下所有生产环境 `.rs` 文件，排除 `tests/`
   和 `fixtures/` 目录。
2. **伴生规格文件** — 与上述源文件同目录的所有 `.liyi.jsonc` 文件。
3. **规格内容** — 从 JSONC 伴生文件中解析：条目名称、`intent` 文本、
   `=doc` / `=trivial` 标记、`reviewed` 状态。

报告中包含三个层级的压缩指标：

| 层级 | 含义 | 包含内容 |
|---|---|---|
| **完整伴生文件**（行/字节比） | 规格文件总量与源代码之比 | JSON 样板、哈希值、行号范围、意图文本 |
| **字节比** | 伴生文件与源代码的原始字节之比 | 同上，按字节度量 |
| **纯意图文本** | 仅自然语言意图字符串 | 仅 `"intent"` 字段值，不含 JSON 结构 |

纯意图文本层级是审查者实际需要阅读的内容。JSON 元数据
（`source_span`、`source_hash`、`tree_path` 等）由检查工具消费，
审查者无需阅读。

## 结果（2026-03-14 快照）

### 汇总指标

| 指标 | 值 |
|---|---|
| 生产源文件数 | 39 |
| 伴生规格文件数 | 38 |
| 源代码行数 | 10,488 |
| 源代码字节数 | 368,695 |
| 伴生文件行数（完整 JSON） | 2,012 |
| 伴生文件字节数 | 97,051 |
| **完整伴生文件 / 源代码（行）** | **19.2%** |
| **完整伴生文件 / 源代码（字节）** | **26.3%** |

### 规格内容分析

| 指标 | 值 |
|---|---|
| 总条目规格数 | 106 |
| 描述性意图 | 102 |
| `=doc` 标记 | 4 |
| `=trivial` 标记 | 0 |
| 已审查（人工确认） | 106（100%） |
| 意图文本总字符数 | 31,500 |
| 平均意图长度 | 309 字符（约 4 行） |
| 每条规格对应的平均源代码行数 | 98.9 |

### 审查压缩

| 指标 | 值 |
|---|---|
| 纯意图文本（按 80 字符/行估算行数） | 约 394 |
| **意图文本 / 源代码行数** | **3.8%** |
| **意图文本 / 源代码字节数** | **8.5%** |
| **缩减倍数** | **26.6×** |

### 按模块统计

| 模块 | 源代码行数 | 规格行数 | 伴生文件比例 |
|---|---|---|---|
| liyi（核心） | 4,280 | 939 | 21.9% |
| liyi-cli | 985 | 248 | 25.2% |
| liyi::tree\_path | 5,209 | 825 | 15.8% |

### 特殊情况

- **check.rs** — 伴生文件比例为 4.4%（1,555 行源代码，68 行规格）。
  最大的模块压缩效果最好，因为其大量 match 分支由少数行为规格覆盖。
- **schema.rs** — 比例为 147.8%（23 行源代码，34 行规格）。一个简单
  的重导出模块，规格元数据超过了源代码——这是针对极小文件的预期边界
  情况。

## 解读

设计文档声称每项的审查成本"约为代码面的 10%"。本次测量得出两个互补
结论：

1. **完整伴生文件比例约 19%** — 包含审查者无需阅读的结构化元数据在内
   的完整 JSON 伴生文件约为源代码的五分之一。这是上界。

2. **纯意图文本约 3.8%** — 审查者实际阅读的自然语言文本不到源代码的
   4%。这是审查面的下界。

设计文档中的"约 10%"落在上下界之间，合理的解释是：该估计涵盖了阅读
意图文本以及浏览结构性上下文（条目名称、行号范围）的成本，但不包括
完整 JSON 元数据。

26.6 倍的压缩因子特定于当前快照下的本代码库。不同代码库的差异：

- 结构重复性高的代码库（语言语法、CRUD 端点、生成的适配器）压缩率
  **更高**。
- 密集且各异的业务逻辑代码库（每个函数都需要详细行为规格）压缩率
  **更低**。

`tree_path` 模块（伴生文件比例 15.8%）展示了结构重复场景：20 个
语言特定文件共享同一结构模式，每种语言的一条意图规格即可覆盖数百行
match 分支。

## 如何复现

```sh
python3 scripts/measure-intent-ratio.py [ROOT]
```

在代码库根目录运行（或将 `.` 作为 ROOT 传入）。仅需 Python 3.10+
及标准库。
